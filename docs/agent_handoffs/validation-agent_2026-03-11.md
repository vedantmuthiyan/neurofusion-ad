## Decisions Made (with rationale)

- **KernelExplainer only (never DeepExplainer)**: SAD-001 §5.5 explicitly requires model-agnostic explanation. KernelExplainer makes no assumptions about model internals, satisfying FDA AI/ML explainability guidance.
- **Module-level shap import with try/except**: allows mock injection via sys.modules for unit tests without needing real shap installed. Local import (`import shap` inside __init__) broke patch-based mocking because the reference was function-local.
- **AttentionAnalyzer uses forward-hook patching (not nn.MultiheadAttention register_forward_hook)**: nn.MultiheadAttention discards attn_weights by default when `need_weights=False`. Patching CrossModalAttention.forward with a wrapper that sets `need_weights=True, average_attn_weights=False` gives per-head [B, H, 4, 4] weights without any model code modification.
- **bootstrap_metric is a public method (not private)**: regulatory requirement — the CI computation logic must be independently testable and auditable per IEC 62304 §5.5.5.
- **temperature scaling uses scipy.optimize.minimize_scalar bounded [0.1, 10.0]**: avoids gradient tape / autograd setup; bounded Brent method is numerically robust; pure numpy/scipy means no torch dependency in calibration module.
- **Fairness threshold 0.07 for max_auc_gap**: taken directly from RMF-001 §5.3; if the threshold is updated in regulatory docs, only SubgroupAnalyzer.MAX_AUC_GAP_THRESHOLD needs to change.
- **NaN-safe regression**: uses `valid_mask = ~np.isnan(mmse_true)` before computing RMSE/MAE/R2; returns NaN (not 0) when all targets are NaN — this is the correct behavior for Bio-Hermes val set where MMSE_SLOPE is always NaN.

## Open Questions for Human Review

- `lifelines` and `scipy` are now runtime dependencies — they should be added to `requirements.txt`. Recommend: `lifelines>=0.30.0`, `scipy>=1.10.0`.
- `shap` library is a runtime dependency for `run_shap.py` and production use. Install with `pip install shap>=0.44.0`. Not required for unit tests (mocked).
- The attention weight patching approach in AttentionAnalyzer mutates `model.fusion.forward` temporarily. This is safe for single-threaded inference but NOT safe in multi-threaded serving (e.g., FastAPI with multiple workers). The API agent should use a separate model instance per request, or a proper hook-based approach with `register_forward_hook` + `need_weights` kwarg if torch version supports it.
- evaluate.py currently uses dry-run mode even when --adni-checkpoint is provided (placeholder `results = _run_dry_run_evaluation()`). The real ADNI CSV loading is stubbed — a follow-up session should wire `src/data/csv_dataset.py` into the evaluate.py pipeline once the CSV dataset loader is confirmed stable.
